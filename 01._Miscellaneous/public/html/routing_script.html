1. Routing with Express Router

As your app grows, putting all your routes in one file gets messy. Express has a built-in Router that lets you split your routes into separate files:


        // routes/pagesRouter.js
        import { Router } from 'express';
        const router = Router();
 
        router.get('/about', (req, res) => {
            res.send(aboutPage);
        });

        export default router;
        

        // app.js
        import pagesRouter from './routes/pagesRouter.js';
 
        app.use(pagesRouter);

The route defined in pagesRouter.js is now accessible at /about
This keeps the code organised. So each router handles its own 'area', like if we had a contactPage, showmorePage etc.

2. Scripts in package.json

Instead of typing out long commands every time, you can define shortcuts in the scripts field of your package.json:


        {
            "scripts": {
                "start-dev": "cross-env PORT=8080 NODE_ENV=DEV node app.js",
                "start-prod": "cross-env PORT=80 node app.js"
            }
        }
        

        npm run start-dev   // sets port to 8080 and NODE_ENV to DEV and runs app with node app.js in development mode
        npm run start-prod  // sets port to 80 and runs node app.js in production mode

dev is used for development mode e.g. with nodemon that automatically restarts the server whenever you save a file.

3. Environment variables and port config

Hardcoding a port number like 8080 can be done using process.env:


        const PORT = process.env.PORT || 8080;
 
        const server = app.listen(PORT, () => {
            console.log("Server started on port", server.address().port);
        });

process.env.PORT checks if the environment has a port set.
If it doesn't find one, it falls back to 8080 which is default
server.address().port logs the port the server ends up running on.
