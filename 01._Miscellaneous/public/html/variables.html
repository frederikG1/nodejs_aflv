Hub Loops Functions

1. Const vs Let

Const is used per default, switch to let if the value could change


            const name = "Frederik"; 
            

            let age = 25;
            

            age = 21; (This works)
            

            name = "Frede" (This doesn't work)

2. Var & scopes

Do not use var per default. It can cause issues through scopes

In this example it shows how var 'bleeds' through the scope, it thinks there is only one 'i' for the whole loop


            for (var i = 0; i < 5; i++) {
                setTimeout(() => {
                console.log(i)
            }, 1000)       
        }
Output: 5 5 5 5 5

In this example it shows how let is bound to the scope, meaning each time {} is iterated it creates a "new" 'i'


            for (let i = 0; i < 5; i++) {
                setTimeout(() => {
                console.log(i)
            }, 1000)       
        }
Output: 0 1 2 3 4
