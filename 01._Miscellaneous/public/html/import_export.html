1. Using express.static()

Instead of writing a route for each file, you can use the built-in express.static() middleware:


        const express = require('express');
        const app = express();
 
        app.use(express.static('public'));

This tells Express to serve any file inside the public folder. So if you have public/html/overview.html, it becomes accessible at /html/overview.html

2. Exporting and importing

Adding type="module" and using import/export makes it possible to split JavaScript into multiple files: 


        // utils.js
        export const greet = (name) => `Hello ${name}`;
 
        // app.js
        import { greet } from './utils.js';
        console.log(greet('Frederik')); // Hello Frederik

Without type="module", we would get an error trying to do this

3. CommonJS vs. ES Modules in Node.js

They do the same thing, just with different syntax:


        // CommonJS — Node default
        const express = require('express');
        module.exports = { something };
 
        // ES Modules — This is more modern but requires "type": "module" in package.json
        import express from 'express';
        export { something };

As stated before, they do the same thing. But some differences are:
CommonJS loads files synchronously and works in Node by default.
ES Modules loads asynchronously and needs "type": "module" in package.json.

4. Client-side vs. server-side redirection

Sometimes you need to send the user to a different page like so:


        // Server-side — the server tells the browser to go to a different page
        app.get('/', (req, res) => {
            res.redirect('/html/overview.html');
        });

        // Also server-side - but using path import
        import path from 'path';
        
        app.get('/', (req, res) => {
            res.sendFile(path.resolve('/html/overview.html'));
        });
    
        // Client-side — the browser navigates to a different page
        window.location.href = '/html/overview.html';

With SSR the server builds the HTML page and sends it to the bwoser
With CSR the server sends HTML + JavaScript and then the browser runs it
